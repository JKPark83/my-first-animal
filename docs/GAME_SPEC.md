# 🦙 알파카 농장 (My First Animal) — 게임 스펙 & 룰 문서

> 멀티플레이 로블록스 알파카 농장 케어·경영 게임의 전체 사양·규칙 정리.
> 본 문서는 실제 코드(`src/`)를 기준으로 작성되었으며, 수치는 모두 `GameConfig.luau` 등 소스의 실제 값입니다.
> 최종 정리일: 2026-06-20

## 1. 게임 개요

| 항목 | 내용 |
| --- | --- |
| 제목 | 알파카 농장 (My First Animal) |
| 장르 | 동물 케어 · 수집 · 방치형 경영 |
| 플랫폼 | Roblox (PC / 모바일 / 콘솔) |
| 모드 | 멀티플레이 (1서버 최대 8농장) |
| 빌드 도구 | Rojo (Luau `--!strict`) |

### 한 줄 요약
빈 농장에 떨어진 플레이어가 **시작 자금 $300**으로 알파카를 사서 자기 농장에서 키우고, **먹이주기·빗어주기·목욕**으로 돌보며 돈을 벌어, **10종 알파카 도감**을 완성해 나가는 게임.

### 핵심 게임 루프
1. **구매** — 가게에서 알파카를 산다 (50달러~300만달러, 희귀도 6단계).
2. **이동** — 내 농장(울타리 구역)으로 데려와 슬롯에 배치한다.
3. **케어** — 시간이 지나면 쌓이는 욕구(배고픔·털엉킴·더러움)를 화면 하단 버튼으로 해소한다.
4. **수익** — 케어할 때마다 즉시 보상, 그리고 행복한 알파카는 양털을 쌓아 패시브 수익을 낸다.
5. **확장** — 번 돈으로 슬롯을 늘리고 더 비싸고 희귀한 알파카를 모은다.
6. **완성** — 도감 10종을 모두 채우면 거액의 완성 보상을 받는다.

## 2. 기술 스택 & 프로젝트 구조

- **언어/타입**: Luau `--!strict`
- **빌드**: Rojo v7.6.1 (`rojo build -o game.rbxlx`)
- **구문 검증**: `luau-ast <file>` (신뢰 가능한 문법 체커)
- **데이터 저장**: `DataStore` (플레이어 영속 저장) + `OrderedDataStore` (리더보드)

### 서버 부팅 순서 (`Main.server.luau`)
실행 순서가 중요하다 — 잘못되면 플레이어 로드 시 농장/알파카가 안 뜬다.
1. **월드 생성**: `FarmBuilder.build()` + `AlpacaPreviewBuilder.init()`
2. **서비스 init** (DataManager보다 먼저 — 훅 등록·Remote 연결):
   FarmService → AlpacaShopService → CareService → WoolService → CollectionService → AlpacaAI → LeaderboardService → GamePassService → TeleportService → VehicleService
3. **DataManager.init()** — **반드시 마지막**. 이때 플레이어 로드가 시작되며, 이미 등록된 `onLoaded` 훅(농장 배정·알파카 스폰)이 동작한다.

### 주요 서버 모듈
| 모듈 | 역할 |

## 5. 케어 시스템

각 알파카에는 3가지 욕구(needs)가 있으며, 값은 0~100이다.

| 욕구 | 케어 액션 | 효과 |
| --- | --- | --- |
| 배고픔 (Hunger) | 먹이주기 (feed) | Hunger → 0 |
| 털엉킴 (Tangle) | 빗어주기 (brush) | Tangle → 0 |
| 더러움 (Dirt) | 목욕 (bath) | Dirt → 0 |

### 욕구 누적 규칙
| 설정 | 값 |
| --- | --- |
| 0→100 도달 시간 (riseTime) | **300초 (약 5분)** |
| 틱 간격 (tickInterval) | 2초 |
| 틱당 증가량 | 약 0.667 |
| 케어 가능 임계치 (careThreshold) | **20 이상**일 때만 케어 가능 |
| 액션 디바운스 | 0.4초 |
| 케어 가능 거리 (proximity) | 16 studs |

- 욕구가 **20 미만이면 케어 불가** → 사실상 케어 한 번 후 약 60초의 자연 쿨다운이 생긴다.
- **시각 경고 임계치**: Hunger ≥ 55, Tangle ≥ 50, Dirt ≥ 50일 때 알파카 머리 위 게이지/표정으로 표시.

### 굶주림 사망
| 설정 | 값 |
| --- | --- |
| 경고 시점 (starveWarnTime) | **600초 (10분)** 먹이 미급여 |
| 사망 시점 (starveDeathTime) | **1200초 (20분)** 먹이 미급여 |

- 마지막으로 먹인 시각(`Fed` 속성) 기준으로 누적 시간을 측정한다. **접속 중일 때만** 계산되며 스폰·로그인·먹이주기 때마다 기준 시각이 갱신돼 오프라인 시간은 제외된다.
- 10분 경고 시 알림을 1회 보내고(`StarveWarned`), 20분까지 먹이를 주지 않으면 알파카가 **영구 사망**한다 → 모델 제거 + 데이터 삭제로 슬롯이 비워진다.
- **먹이주기(feed)** 액션은 배고픔을 0으로 만드는 동시에 이 굶주림 타이머도 리셋한다.

### 케어 동작(모션)
- **빗질·목욕**은 캐릭터가 팔을 움직이는 절차적 동작을 취한다(R15 어깨 `Motor6D`를 매 프레임 덮어씀, 빗질=한 팔·목욕=양 팔). R6 캐릭터는 기존 이모트로 대체된다.
- **먹이주기**는 기존 이모트를 사용하며, 먹이주기 시 진행 중이던 빗질/목욕 동작은 즉시 취소된다.

### 행복도 (Happiness)
```
Happiness = clamp(100 − (Hunger + Tangle + Dirt) / 3, 0, 100)
```
욕구가 낮을수록 행복하고, 행복도는 양털 수익에 직접 영향을 준다.

## 6. 수익 시스템

수익은 **① 케어 즉시 보상**과 **② 양털 패시브 수익** 두 갈래다.

### ① 케어 즉시 보상
케어 액션 1회 성공 시 지급:
```
지급액 = floor( careReward × 게임패스배수 × moneyMultiplier(5) )
```
예) 들풀(careReward 3) 케어 1회 = 3 × 5 = **$15** (게임패스 보유 시 ×2 = $30).

### ② 양털 패시브 수익
행복한 알파카는 시간이 지나면 양털을 쌓고, 플레이어가 수동으로 수집한다.

| 설정 | 값 |
| --- | --- |
| 적립 주기 (cycle) | 45초 |
| 자동 수집 | ❌ (수동 — 양털 수집 버튼) |

**적립 공식 (45초마다):**
```
적립량 = woolValue × (0.5 + 0.5 × 행복도배수) × moneyMultiplier(5)
```
- 행복도 100이면 배수 1.0 → 주기당 `woolValue × 5` 적립.
- 행복도 0이면 배수 0.5 → 주기당 `woolValue × 2.5` 적립.

**누적 상한 (cap):**
```
누적 상한 = woolValue × capCycles(240)
```
> ⚠️ 주의: **상한에는 ×5(moneyMultiplier)가 곱해지지 않는다.** ×5는 적립 *속도*에만 적용되고, 한 알파카가 쌓을 수 있는 양털 총량은 `woolValue × 240`에서 멈춘다.

**수집:**
```
수집액 = floor( 누적 양털 × 게임패스배수 )
```
양털 수집 버튼(우상단 아이콘)을 누르면 보유한 모든 알파카의 양털을 한 번에 거둔다.

## 7. 도감 · 순위 · 진행

### 도감 (Collection)
- 처음 보유하는 종마다 도감에 기록된다 (총 10종).
- **도감 완성 보상**: 10종을 모두 모으면 **$1,000,000** 지급 (세션당 1회).

### 리더보드 (총자산 순위)
| 설정 | 값 |
| --- | --- |
| 표시 인원 | 상위 10명 |
| 캐시 주기 | 60초 |
| 제출 주기 | 60초 |
| 저장소 | `OrderedDataStore "AlpacaWorth_v1"` (키 `u_{userId}`) |

**총자산(netWorth) 계산:**
```
netWorth = 보유 머니 + Σ(알파카 가격) + Σ(미수집 양털)
```
순위판은 농장 구역의 큰 보드(좌표 약 (48, 13, −10))에 표시된다.

## 9. 도로망 · 탈것 (카트)

월드를 더 풍성하게 꾸미기 위해 도로망과 탈 수 있는 카트를 추가했다.

### 도로망 (`FarmBuilder`)
- **중앙 대로**: 폭 약 67, 길이 92 (z ≈ 21~113), 도로 윗면 y=0.40.
- **전면 도로**: 폭 16, 길이 640 (농장 앞을 가로지름).
- **인도/연석**, **차선 표시**.
- **가로등**: x=±13.5, z=24~108 (28 간격), PointLight(Range 24, Brightness 1.4).
- **가로수 6그루**: (−34,44), (34,60), (−34,88), (34,30), (−60,70), (60,96).
- **허브 경사로 (HubRamp)**: Size(22, 0.6, 12), Pos(0, 0.375, 29), 2.6° 기울기 — 허브 데크(y=0.95)와 중앙 대로(y=0.40)의 단차를 매끄럽게 연결.

### 농장 카트
차고 패드(GaragePad)를 밟으면 귀여운 카트가 소환되어 자동 탑승한다.

| 설정 | 값 |
| --- | --- |
| 최고 전진 속도 | 62 |
| 가속 반응성 | 50 |
| 회전 속도 | 2.6 |
| 후진 속도 | 최고속의 40% |
| 정지 임계 | 속도 0.5 미만이면 정지 |
| 하차 | 점프(Space) |
| 차고 쿨다운 | 3초 (재소환 방지) |

- 차체는 `VehicleSeat`(충돌 본체) + 후드/범퍼/헤드라이트/등받이/롤바/트렁크/테일라이트/바퀴 4개(장식, 충돌 없음).
- **주행 물리는 클라이언트(`VehicleController`)가 담당** — 시트의 ThrottleFloat/SteerFloat(WASD) 입력을 읽어 `AssemblyLinearVelocity`/`AssemblyAngularVelocity`로 직접 제어. 그래서 카트 소환 시 네트워크 소유권을 운전자에게 넘긴다.
- 1인 1카트 — 새로 소환하면 기존 카트는 제거되고, 퇴장·사망·리스폰 시에도 정리된다.
- 카트 위에 "🛺 (플레이어명)" 이름표 표시 (최대 거리 200 studs).

## 15. 이번 작업(개선) 요약

이번 세션에서 추가한 4가지 기능:

1. **악세사리 가게** — 알파카 옷 8종(밀짚모자~왕관)을 파는 전용 가게. 구매하면 보유 목록에 쌓이고, 악세사리 하나를 고른 뒤 농장의 특정 알파카를 선택해 입히거나 벗길 수 있다. 가게로 순간이동하는 `🎀 악세사리` 버튼을 제공하며, 악세사리 파츠는 알파카 모델에 부착돼 이동·AI 동작을 그대로 따라간다.
2. **먹이 상점(인벤토리형)** — 먹이 5종(풀·당근·건초·프리미엄·매직)을 판매하며, 종마다 가격·케어 보상 배수·정돈 효과·양털 보너스가 다르다. 먹이를 사면 개수가 쌓이고, '사용 먹이'를 고르면 먹이주기마다 1개씩 소비되며 재고가 떨어지면 무료 기본 먹이(풀)로 자동 대체된다. `🍽️ 먹이상점` 순간이동 버튼 제공.
3. **굶주림 사망** — 먹이를 오래 주지 않으면 알파카가 떠난다. 접속 중 누적 시간 기준으로 10분(`starveWarnTime`)에 경고하고 20분(`starveDeathTime`) 동안 먹이를 주지 않으면 영구 사망해 슬롯이 비워진다. 스폰·먹이주기 시 기준 시각(`Fed`)을 갱신해 오프라인 시간은 계산에서 제외한다.
4. **빗질·목욕 동작** — 빗질/목욕 시 캐릭터가 실제로 팔을 움직인다. R15 어깨 관절(`Motor6D.Transform`)을 매 프레임 덮어쓰는 절차적 애니메이션이며(빗질=한 팔, 목욕=양 팔), R6 캐릭터는 R15 팔이 없어 기존 이모트로 대체된다.

이후 적대적 코드리뷰로 발견한 보강 사항: 먹이 소진 시 사용 먹이 자동 복귀, 미보유 유료 먹이 선택 차단, 먹이주기 시 진행 중이던 빗질/목욕 동작 즉시 취소, 퇴장 시 디바운스 항목 정리, 세 상점 팝업 모두 패드 이탈 시 자동 닫힘.

## 16. 알려진 제한 사항 / 향후 과제

- **슬롯 11·12번째 비용**($12M/$25M)은 정의돼 있으나 `maxSlots=10`이라 도달 불가(미사용).
- **양털 개별 수집** 로직은 서버에 구현돼 있으나, 현재 클라이언트 UI는 항상 "전체 수집"만 호출한다.
- **수익화 전면 비활성** — 게임패스/로벅스 알파카는 정식 출시 시 ID 연결 필요.
- **배경음악**은 플레이스홀더 트랙 1개.
- `GameConfig.Sounds.eat` 키는 구(舊) 키로 현재 미사용(=`feed`와 동일).
